# Design System — PPT 社媒拼图模板设计器

## Product Context

- **产品是什么：** 一个本地运行的 PPT 社媒拼图模板编辑器，用来定义多张社媒画布、PPT 页框、背景效果和可迁移模板包。
- **用户是谁：** 需要把 PPT 内容快速改造成视频号、小红书、抖音等竖版或方图素材的内容创作者、运营、设计助理和自动化工作流维护者。
- **行业空间：** 内容生产工具、模板编辑器、本地创作工作台。
- **项目类型：** 工具型 Web App。第一屏就是编辑器，不做营销页。

## Aesthetic Direction

- **方向：** Industrial / Utilitarian，工业化工具界面。
- **装饰级别：** 克制但有质感。界面主要依靠网格、面板、状态色、排版密度建立秩序。
- **气质：** 像一个可靠的模板工作台，而不是海报生成器官网。用户打开后应该立刻知道哪里选画布、哪里调参数、哪里验证模板。
- **不采用：** 大面积紫蓝渐变、营销英雄区、大圆角卡片堆叠、三栏功能卖点、装饰性光斑背景。

## Typography

- **中文 UI / 正文：** `"PingFang SC", "Noto Sans SC", "Source Han Sans SC", system-ui, sans-serif`
  - 理由：中文标签密集，必须清楚、熟悉、低噪音。本地工具不依赖外部字体也能稳定显示。
- **英文 / 数字 / 坐标：** `"JetBrains Mono", "SFMono-Regular", "Menlo", monospace`
  - 理由：页码、尺寸、坐标、schema 状态需要等宽数字，便于扫描和对齐。
- **标题：** 使用同一中文 UI 字体，加粗，不引入额外展示字体。
- **加载策略：** 首版使用系统字体，避免本地离线启动时字体资源失败。若未来需要品牌化，再自托管 `Noto Sans SC` 和 `JetBrains Mono`。
- **字号尺度：**
  - 页面标题：24px / 32px，700
  - 面板标题：14px / 20px，700
  - 控件标签：12px / 18px，700
  - 正文：14px / 22px，400
  - 状态栏 / 元信息：12px / 18px，等宽或半等宽

## Color

- **策略：** 平衡型。冷中性色做工作台，青绿色做操作焦点，语义色只用于状态。
- **Primary / 主操作：** `#0B4F71`
  - 用于导出、确认、当前选中边框、关键按钮。
- **Accent / 编辑焦点：** `#00A6A6`
  - 用于选中框、拖拽锚点、当前画布强调、可交互焦点。
- **Surface / 页面背景：** `#EEF1F2`
  - 比当前偏米色背景更冷静，减少“老式后台”感。
- **Panel / 面板背景：** `#F8F6F0`
  - 保留轻微暖色，让长时间编辑不刺眼。
- **Canvas Grid / 工作区网格：** `#D5DCE0`
- **Ink / 主文字：** `#10202B`
- **Muted / 次级文字：** `#60717C`
- **Border / 分割线：** `#BCC7CC`
- **Success：** `#1F7A4D`
- **Warning：** `#C97A24`
- **Error：** `#B84242`
- **Info：** `#0B6BCB`
- **Dark mode：** V1 不做深色模式。设计上预留 token，但编辑画布类工具优先保证浅色环境下颜色判断准确。

## Spacing

- **基础单位：** 4px。
- **密度：** Compact，偏紧凑。这个产品是反复操作的编辑器，不是展示页。
- **尺度：** 2xs 2px, xs 4px, sm 8px, md 16px, lg 24px, xl 32px, 2xl 48px。
- **面板内间距：** 16px。
- **控件间距：** 10px 到 14px。
- **工具栏高度：** 48px 到 56px。

## Layout

- **方式：** Grid-disciplined，严格三栏编辑器。
- **桌面布局：**
  - 左栏：260px，画布列表和批量操作。
  - 中区：自适应，Konva 画布和工具条。
  - 右栏：340px，属性检查器。
  - 底部：36px 到 44px，模板合同健康状态。
- **移动端：** V1 不作为主场景。窄屏可以堆叠为画布优先、面板折叠，但不牺牲桌面工作效率。
- **圆角：**
  - 控件：6px
  - 重复项 / 画布条目：8px
  - 弹窗：10px
  - 画布本身不加大圆角，避免误导真实导出边界。

## Components

- **顶部栏：** 显示产品名、模板名输入、导入导出操作。全部中文。
- **画布列表：** 每项显示画布名称、比例、绑定页码摘要、缺失素材状态。当前画布使用深色底或左侧高亮条。
- **中间画布：** 保留网格背景，增加缩放比例、适配窗口、当前尺寸、添加 PPT 框。
- **右侧属性面板：** 根据选择状态切换。
  - 未选中框：画布设置、背景设置、背景效果。
  - 选中框：页码、位置尺寸、适配方式、透明度、圆角、阴影、边框。
- **批量创建：** 不使用浏览器 prompt。改为弹窗或面板内表单，实时显示“将创建 N 张，页码每张 +M，最终总数 X/18”。
- **底部状态栏：** 固定显示 schema 校验、素材完整性、保存状态、最近操作。

## Interaction

- **拖拽框：** 拖拽结束后吸附到整数坐标。后续可加 8px 网格吸附开关。
- **缩放框：** PPT 框保持 16:9，只显示四角锚点，不允许旋转。
- **页码输入：** 只允许大于 0 的整数。非法输入立即恢复或提示。
- **批量创建：** 超过 18 张时禁用确认按钮，不等提交后才报错。
- **导入 ZIP：** 解析中显示状态；绝对路径、`..` 路径、缺失 assets 时给中文错误。
- **导出 ZIP：** 导出前跑 schema 校验和资产完整性检查。
- **空状态：** 没有 PPT 框时，中间画布不显示说明文字，只在工具条提供清楚的“添加 PPT 框”按钮。

## Motion

- **方式：** Minimal-functional。
- **时长：**
  - hover / focus：80ms 到 120ms
  - 面板切换：150ms
  - 弹窗进入：180ms
- **原则：** 动效只服务状态变化，不做装饰。编辑器里过度动效会降低精确感。

## Prototype Adjustments

当前原型不需要推翻，但需要升级成中文生产工具：

1. 全部界面文案中文化，保留 JSON、ZIP、schema 等必要技术名词。
2. 右侧属性面板按“画布 / 背景 / 框 / 效果”分组。
3. 批量创建从 `window.prompt` 改成结构化表单。
4. 中间工具条增加缩放、适配窗口、画布比例和样张验证入口。
5. 底部状态栏改为更可读的中文合同健康条。
6. 控件增加明确 focus、disabled、active 状态。
